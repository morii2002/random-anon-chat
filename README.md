# ChatPat - 匿名ランダムチャット(プロトタイプ)

テキストベースの匿名1対1ランダムマッチングチャットです。Node.js + WebSocket (ws) + Express で構成。

## 構成

```
.
├── server.js        # WebSocketサーバー(マッチング・中継ロジック)
├── package.json
└── public/
    ├── index.html   # 画面(スタート画面・チャット画面・広告枠)
    ├── style.css
    └── script.js    # クライアント側ロジック
```

## ローカルでの起動方法

```bash
npm install
npm start
```

`http://localhost:3000` をブラウザで2タブ以上開くと、ランダムにマッチングされてチャットできます。

## 実装済み機能

- 匿名・名前登録不要での接続
- ランダムマッチング(待機キューから2人組を作成)
- テキストチャット(1000文字制限)
- 「入力中...」表示
- スキップ(別の相手と再マッチング)
- 終了(キューから抜ける)
- 通報ボタン(サーバーログに記録するのみの簡易実装)
- 利用ルールへの同意チェック(18歳未満禁止の明記)
- 広告枠(上部・サイドバー・下部)とアフィリエイトリンク枠をUIに用意

## マネタイズ(広告・アフィリエイト)の組み込み方

`public/index.html` 内の以下の要素に、各広告ネットワークのタグやアフィリエイトリンクを差し込みます。

- `.ad-top` / `.ad-bottom` / `.ad-side` : Google AdSense や A8.net などの広告タグを設置
- `.affiliate-box` 内の `<a class="aff-link">` : アフィリエイトリンクのURLとテキストを差し替え

広告審査(特にAdSense)では、運営者情報・プライバシーポリシー・利用規約・問い合わせ先の明記が求められることが多いので、別途ページを用意してください。

## 本番運用前に必ず検討すべき事項(重要)

匿名ランダムチャットサービスは、過去に未成年保護や違法コンテンツの流通が大きな問題となり、サービス停止に至った例(Omegleなど)があります。マネタイズして公開運用する場合、以下は必須レベルで検討してください。

1. **年齢確認・年齢制限の強化**: チェックボックスだけでは不十分。利用規約上の年齢制限の明記、可能であれば年齢確認手段の導入。
2. **モデレーション**: NGワードフィルタ、画像/動画を扱う場合は自動検出(CSAM対策含む)、通報内容の人による確認体制。
3. **ログ・通報の保存**: 違法行為があった場合に捜査機関へ協力できるよう、最低限の接続ログ(IP、タイムスタンプ等)を一定期間保持する設計。
4. **利用規約・プライバシーポリシー**: 個人情報の扱い、禁止行為、通報・凍結のルールを明文化。
5. **法令対応**: 出会い系サイト規制法(日本国内向けの場合、年齢確認義務など)に該当する可能性があるため、サービス内容によっては弁護士への確認を推奨します。
6. **インフラ**: 本プロトタイプはメモリ上で状態管理しているため、複数サーバーでスケールする場合はRedis等での状態共有が必要。

このプロトタイプはあくまで動作確認用です。公開・収益化する前に上記の対応を進めてください。

## デプロイ

Node.js + WebSocketが動く環境(Render, Railway, Fly.io, VPSなど)であれば動作します。Vercelなどのサーバーレス環境は永続的なWebSocket接続に向かないため非推奨です。
